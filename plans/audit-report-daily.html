Finding 1 — why February breaks (data flow)

producer            consumers (all expect the dir spelling)
─────────           ────────────────────────────────────────
getDailyDateToday   ──"february"──▶  page.tsx  ─▶ getDaily()  ─▶ import('@/content/february/N.mdx')  ✗ not found → crash
  toLocaleString()                  opengraph-image.tsx ─▶ getDaily() ─▶ same crash

getMonthToday       ──"febuary"───▶  /[month] pages, calendar nav                                  ✓ works
  monthSchema.options[getMonth()]

content dir:  src/content/febuary/   ·   monthSchema: [... "febuary" ...]   ·   localStorage keys: __daily_/febuary/

FIX (plan 010): make getDailyDateToday derive the month the SAME way getMonthToday does —
monthSchema.options[today.getMonth()] — so it emits "febuary". No dir rename → saved likes & shared URLs stay intact.

A prior audit logged this as F8 and deprioritized it ("the app works, all three agree") — it missed that getDailyDateToday's runtime output is a fourth value that disagrees. Plan 010 supersedes F8 with a no-rename fix.
