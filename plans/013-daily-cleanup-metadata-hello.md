## Why this matters

Two small loose ends ship to production:

1. **`/hello` debug route** — `src/app/hello/page.tsx` renders
   `new Date().toString()` and is `force-dynamic`. It is a leftover scratch route
   reachable at `https://daily.orlan.dev/hello`.
2. **Placeholder root metadata** — `src/app/layout.tsx` still exports the
   create-next-app defaults: title `"Create Next App"`, description
   `"Generated by create next app"`, and sets **no** `metadataBase`. So the
   site's default/fallback `<title>` is wrong, and OpenGraph/Twitter image and
   URL resolution has no base origin to resolve against. The app's real identity
   already exists in `manifest.ts` ("Daily Stoic", "366 Meditations on Wisdom,
   Perseverance, and The Art of Living") and `BASE_URL` in `lib/constants.ts`.

Per-page routes already set their own titles/descriptions (`page.tsx`,
`[month]/page.tsx`, `[month]/[day]/page.tsx`), so this only fixes the
app-wide fallback and base URL — it does not change those per-page values.

> Note: the `title.template` adds `" — Daily Stoic"` to pages that set a string

> the tab/SEO title. If you judge the template undesirable for those pages, use a
> plain `title: "Daily Stoic"` string instead and note the choice — either is fine.
