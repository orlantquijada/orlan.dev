## What was NOT turned into plans (from the audit)

Carried as findings in `audit-report.html`, not selected for planning this round:

- **F5** — `getDaily`/`getValidDates` uncached; homepage re-parses MDX twice per
  request. Wrap in React `cache()`. (perf, S–M, MED risk re: daily rollover.)
- **F6** — daily API route (`api/[month]/[day]/route.ts`) skips input validation
  and echoes params in its 404. NOT path traversal (the dynamic `import()` is
  bundler-scoped). Minor hardening: mirror the pages' `isValidDate` guard, static 404. (security, S, LOW severity.)
- **F7** — 56 transitive dependency advisories (1 critical / 30 high), **all
  build-time** (astro-icon → @iconify/tools; @astrojs/vercel → nft; esbuild
  dev-server-on-Windows = N/A on macOS). Clear via pnpm overrides if CI starts
  running `pnpm audit`. (deps, S–M, LOW runtime risk.)
- **F8 — CORRECTED & now planned as 010.** `"febuary"` is misspelled across the
  month enum, subjects map, and `src/content/febuary/` directory. The earlier note
  said "the app works (all three agree)" — that was **wrong**: `getDailyDateToday`
  (`lib/utils.ts:43`) produces the *correct* spelling `"february"` at runtime via
  `toLocaleString`, which disagrees with the three static spellings, so the home
  page (`/`) and the root OG image **crash every February**. Plan 010 fixes the
  live bug by deriving the month from `monthSchema.options` (no dir rename → no
  orphaned likes, no broken URLs). The full spelling-correction rename (with a
  localStorage key migration + URL redirects) remains deferred. (correctness, HIGH.)
- **F9** — README lists packages that don't exist (`eslint-config-custom`,
  `tsconfig`); only `@repo/utils` + `packages/ui/*` exist. One-line doc fix. (docs, S.)
